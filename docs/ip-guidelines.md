# IP Guidelines

This document is practical project hygiene, not legal advice.

## Recommended Position

Potty Pixel Pal should be an original open-source project inspired by a general idea: retro LCD potty-training toys and paper-roll electronic gadgets.

The project can mention inspiration in documentation, but the repository, assets, app title, and user-facing brand should not present themselves as an official recreation of Smarty Pants, Toy Story, Pixar, or Disney.

## Avoid

- Do not use official screenshots, posters, press images, logos, audio, or video.
- Do not use `Disney`, `Pixar`, `Toy Story`, or `Smarty Pants` in the repository name, package name, app title, or primary branding.
- Do not trace or copy exact UI layout from official images.
- Do not copy movie dialogue or voice lines.
- Do not use official character art as a reference layer in committed files.
- Do not sell merchandise or publish paid builds using protected names or likenesses.

## Safer Pattern

Use wording like:

```text
An original fan-inspired LCD potty-training toy prototype.
Inspired by retro electronic toys and public discussion around Smarty Pants from Toy Story 5.
Not affiliated with Disney, Pixar, Toy Story, or any related rights holders.
No official artwork, screenshots, logos, audio, or proprietary assets are used.
```

## Why This Matters

Non-commercial fan work can still receive copyright or trademark complaints. The most likely outcomes are:

- A request to rename or remove assets.
- A DMCA takedown notice.
- Repository disablement until disputed or fixed.
- A cease-and-desist letter for high-visibility projects.

The safest open-source posture is to make the code, art, names, and sound assets original.

## Asset Rules

Every committed asset should be one of:

- Created specifically for this project.
- Generated with rights compatible with open-source release.
- Public domain or permissively licensed, with attribution.
- A simple geometric shape or icon drawn from scratch.

Document asset sources before adding them.

## Naming Rules

Good:

- `potty-pixel-pal`
- `lcd-potty-pal`
- `paper-roll-pal`
- `potty-pop-lcd`

Avoid:

- `smarty-pants`
- `toy-story-smarty-pants`
- `pixar-potty-toy`
- Any name that implies official association.
